# TasteMap 프로젝트 개요

TasteMap은 내가 직접 방문한 맛집 기록을 지도 위에 시각화하고, 이를 웹에서 지도 형태로 탐색·추천·공유할 수 있게 만드는 서비스다. 사용자는 회원가입/로그인을 통해 자신만의 맛집 데이터를 계속 누적할 수 있으며, 누적된 데이터는 개인 기록을 넘어 AI 기반 추천(RAG) 기능으로 확장된다.

서비스의 핵심 가치는 다음과 같다.

- 검증된 데이터: “가본 곳” 중심으로 신뢰도 있는 기록/평가를 제공  
- 지도 기반 탐색: 텍스트 리스트가 아니라 실제 위치 기반으로 직관적으로 탐색  
- 개인화: 유저가 추가한 데이터가 쌓일수록 추천과 탐색 경험이 고도화  
- 추천 자동화: 유저의 맛집/평가 데이터를 기반으로 AI가 상황별 추천을 제공  
---

# 필요한 기능 상세

## 1) 회원 시스템 (가입/로그인/계정)
사용자가 자신의 맛집 데이터를 소유하고 관리할 수 있도록 계정 기반으로 동작한다.

- 회원가입(이메일/비밀번호 또는 소셜 로그인)
- 로그인/로그아웃
- 내 프로필(닉네임 등 기본 정보)
- 권한 규칙:  
  - 사용자가 등록한 맛집/리뷰는 작성자만 수정·삭제 가능  
  - 관리자 기능(선택): 신고 처리, 블라인드 등  

---

## 2) 지도 기반 맛집 탐색
지도 화면에서 맛집을 핀/마커 형태로 표시하고, 사용자는 위치 기준으로 탐색한다.

- 지도 화면에서 현재 화면(영역) 내 맛집 목록 조회
- 지도 줌/이동에 따라 맛집 마커 동기화
- 내 위치 기반 주변 맛집 조회(반경 기준)
- 맛집 클릭 시 상세 정보 패널/페이지 표시

---

## 3) 맛집 등록/관리 (Place)
사용자가 자신만의 맛집을 지도에 추가하고 관리하는 기능이다.

- 맛집 등록
  - 이름, 카테고리(예: 한식/일식/카페 등)
  - 위치(지도 클릭으로 좌표 지정)
  - 주소/메모/태그(선택)
  - 방문일(선택)
- 맛집 수정/삭제
- 맛집 공개 범위(선택)
  - “나만 보기 / 공개” 같은 visibility 설정을 두면 커뮤니티 확장 시 유리

---

## 4) 평가/리뷰 시스템 (Review)
TasteMap의 “추천”과 “신뢰”를 만드는 핵심 데이터 층이다.

- 평점(예: 1~5점)
- 리뷰 텍스트(추천 이유, 메뉴, 분위기, 재방문 의사 등)
- 추천 표시(예: “재방문/강추/비추” 같은 간단한 플래그 가능)
- 리뷰 수정/삭제
- 맛집 단위 집계 정보(평균 평점, 리뷰 수) 표시

---

## 5) 검색/필터(필수에 가까운 확장)
지도의 탐색 효율을 높인다.

- 키워드 검색(맛집명/메모/태그)
- 카테고리 필터
- 평점 필터
- “내가 추가한 맛집만 보기” 토글
- 정렬(거리순/평점순/최근 추가 순)

---

## 6) AI 맛집 추천 (RAG 기반)
TasteMap의 차별 기능. 사용자가 누적한 맛집 데이터(등록 정보 + 리뷰/평가 + 태그/메모)를 기반으로, AI가 상황별 추천을 제공한다.

### 6-1. 추천 입력(유저가 요청하는 방식)
- 자연어 질의 예:
  - “오늘 강남 근처에서 혼밥 추천해줘”
  - “매운 음식 땡기는데 내가 저장한 곳 중에 추천”
  - “데이트하기 좋은 분위기 있는 곳”
- 조건 기반 추가 입력(선택):
  - 위치(현재 위치/특정 지역)
  - 동행 유형(혼밥/데이트/회식)
  - 예산, 음식 취향, 거리 제한

### 6-2 정보게이지
- 정보 게이지를 만들어서 정보 게이지가 부족할 경우 유저에게 질의를 통해 양질의 답변을하기 위해 정보를 유도
- 예시
- 유저: "오늘 저녁 추천해줘"
- AI: "알겠어. 그 전에 몇가지 질문을 할게. 오늘 점심 뭐먹었는데?" 
- 유저: "짜장면"
- AI: "그럼 저녁은 면말고 밥으로 먹자"

### 6-3. 추천 결과(서비스가 제공해야 하는 형태)
- 추천 맛집 리스트(여러 개)
- 지도 하이라이트(추천된 장소를 지도에서 바로 보여주기)
- “추천이 별로였어요/좋아요” 피드백(선택)

### 6-4. 데이터 범위(프라이버시 관점)
- 기본 원칙: 본인(해당 유저)이 추가한 맛집 데이터만 사용
- 확장 옵션: 공개 맛집까지 포함하는 “커뮤니티 기반 추천” 모드(공개 범위 설정 필요)

---

## 7) 운영/품질 관리(선택, 확장 기능)
커뮤니티 기능이 붙을수록 중요해진다.

- 신고 기능(맛집/리뷰)
- 블라인드/숨김 처리(관리자)
- 스팸/중복 맛집 처리(선택)
- 로그/히스토리(추천 요청 기록, 사용자 활동 기록)
